## 🧠 Strategy

**Moving Average Crossover** — a classic, well-tested trend-following strategy:

| Signal | Condition |
|--------|-----------|
| 🟢 **BUY**  | SMA(20) crosses **above** SMA(50) + RSI < 70 (not overbought) |
| 🔴 **SELL** | SMA(20) crosses **below** SMA(50) + RSI > 30 (not oversold) |
| 🛑 **Stop Loss** | Position drops **-5%** → auto-exit |
| 🎯 **Take Profit** | Position gains **+12%** → auto-exit |

**Universe traded:** SPY, QQQ, IWM, VTI, XLK, XLV, XLE, AAPL, MSFT, GOOGL, AMZN, NVDA, JPM

**Position sizing:** 18% of portfolio per trade, max 5 concurrent positions, 10% cash buffer always maintained.

## 🚀 Quick Setup (15 minutes)

### Step 1 — Create a Free Alpaca Account
1. Go to **[alpaca.markets](https://alpaca.markets)** and sign up (free)
2. UK residents are fully supported
3. Start in **Paper Trading** mode (fake money, real market data) to test first

### Step 2 — Get Your API Keys
1. Log in to Alpaca → Click **"Paper Trading"** in the top left
2. Go to **API Keys** → **Generate New Key**
3. Save your **API Key ID** and **Secret Key** (you won't see the secret again)

### Step 3 — Fork This Repository
Click the **Fork** button at the top of this page to copy it to your account.

### Step 4 — Add GitHub Secrets
In your forked repo: **Settings → Secrets and variables → Actions → New repository secret**

Add these 3 secrets:

| Secret Name | Value |
|-------------|-------|
| `ALPACA_API_KEY` | Your Alpaca API Key ID |
| `ALPACA_SECRET_KEY` | Your Alpaca Secret Key |
| `ALPACA_BASE_URL` | `https://paper-api.alpaca.markets` |

### Step 5 — Enable Actions
1. Click the **Actions** tab in your repo
2. Click **"I understand my workflows, go ahead and enable them"**
3. The bot will now run automatically twice a day!

### Step 6 — Test It Manually
1. Go to **Actions → 🤖 Trading Bot → Run workflow**
2. Click the green **Run workflow** button
3. Watch the logs — you should see it scanning signals

## 💰 Going Live with Real Money

> ⚠️ **Only do this after testing on paper trading for at least 2–4 weeks**

1. Fund your Alpaca account (minimum $50 / ~£40)
2. Switch to Live trading in Alpaca dashboard
3. Generate **new Live API keys** (separate from paper keys)
4. Update your GitHub secrets:
   - `ALPACA_API_KEY` → your live key
   - `ALPACA_SECRET_KEY` → your live secret
   - `ALPACA_BASE_URL` → `https://api.alpaca.markets`

Open `dashboard/index.html` in a browser to see your bot's performance.

3. Branch: `main`, folder: `/dashboard`

## 🛠 Troubleshooting

**Bot runs but makes no trades?**
- MA crossover signals are rare by design — this is normal. The bot may only trade a few times a month.
- Check the logs in Actions → latest run → bot-logs artifact.

**"Account not active" error?**
- Log in to Alpaca and check your account status.

**Orders rejected?**
- Alpaca requires a minimum order value. With £50, some higher-priced stocks may be skipped.
- The bot will log `"insufficient cash buffer"` and skip those.

**Module not found errors?**
- Check that `requirements.txt` is present and the install step ran successfully in Actions.
